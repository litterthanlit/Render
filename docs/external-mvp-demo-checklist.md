# Render External MVP Demo Checklist

Use this flow for a short external demo of the Render MVP. Keep the story focused on one idea: designers learn to build real software by moving from visual intent to shipped, review-ready product work.

## Demo Flow

1. Open `/`.
2. Explain the promise: **Design. Code. Ship.**
3. Show the hero mockup: code, live preview, validation, and product polish in one learning surface.
4. Open `/tracks`.
5. Show the full 14-phase path and explain progression gates.
6. Open an early HTML/CSS lab and show Run, Check, Reset, and Hints.
7. Jump to a React preview lab and show live code plus isolated preview.
8. Jump to Design Systems & Tokens and show tokens/component-system learning.
9. Jump to APIs, Data & State Management and explain loading, empty, error, search, and forms.
10. Jump to Deployment & Production Workflow and show release/readiness framing.
11. Jump to Capstone Project and show the milestone structure.
12. Jump to Portfolio & Career Preparation and show final packaging/readiness.
13. Seed full completion with the dev-only progress helper.
14. Return to `/tracks` and show 100% completion.

## Talking Points

- Render is not a generic coding bootcamp. It teaches designers through components, states, tokens, systems, product flows, shipping, and portfolio proof.
- Labs provide immediate feedback, but projects require reflection and review-ready artifacts.
- The MVP uses local progress. Backend accounts, mentor review, uploads, analytics, Storybook, and real integrations are intentionally deferred.
- The capstone and career phases do not promise employment. They help learners package credible evidence.

## Demo Prep

- Run `npm run build`.
- Run `npm run start` for a production-like smoke test.
- Use the dev-only progress helper in local development when you need to seed milestones quickly.
- Confirm the production build does not show the dev-only progress helper.
